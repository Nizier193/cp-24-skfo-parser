# Получение статуса ответов на отзывы

Метод позволяет получить статус модерации ответов на отзывы или получить
информацию на отдельно взятый ответ на отзыв.

Ограничения:

  * по умолчанию на каждой странице отображается по 20 отзывов. Вы можете отображать до 100 отзывов на странице, указав значение в параметре size;
  * совместно использовать параметры пагинации и фильтрации по 1 ответу на отзыв — нельзя (либо пагинация с указыванием пары параметров page и size либо id).

## Параметры взаимодействия​

`GET`

`https://public-
api.rustore.ru/public/v1/application/{packageName}/feedback?id={id}&page={number}&size={size}`

Атрибут| Тип| Обязателен| Расположение| Описание| Пример  
---|---|---|---|---|---  
`Public-Token`| string| Да| header| [Токен доступа к API RuStore](/help/work-
with-rustore-api/api-authorization-token).| N/A  
`packageName`| string| Да| path| Наименование пакета приложения. Максимальная
длина — 50 символов.| `com.myapp.example`  
`id`| number| Нет| query| `id` конкретного ответа на отзыв, задаётся при
желании получить информацию о конкретном ответе на отзыв.| `743103`  
`page`| number| Нет| query| Номер страницы. Начинается с 0.| 0  
`size`| number| Нет| query| Количество отзывов на странице. По умолчанию — 20
Максимально — 100.| `743103`  
  
## Параметры ответа​

Атрибут| Тип| Обязателен| Описание| Пример  
---|---|---|---|---  
`code`| string| Да| Код ответа.| `error`/`OK`  
`message`| string| Нет| Расшифровка кода ответа.| N/A  
`timestamp`| timestamptz| Да| Время ответа.|
`2022-07-08T13:24:41.8328711+03:00`  
`body{}`| object| Да| N/A| N/A  
  
### body​

Атрибут| Тип| Обязателен| Описание| Пример  
---|---|---|---|---  
`id`| number| Да| `id` ответа на отзыв.| `748479`  
`commentId`| number| Да| `id` отзыва.| `748223`  
`text`| string| Да| Текст ответа.| `Это ответ на отзыв пользователя.`  
`status`| string| Да| Статус ответа на отзыв.| • `PUBLISHED` — опубликован все
ок;  
• `MODERATION` — на модерации;  
• `REJECTED` — отклонён модерацией;  
• `DELETED` — удалён разработчиком или изменён.  
`date`| string| Да| Дата и время последнего изменения| `2023-06-01
18:10:43.043`
