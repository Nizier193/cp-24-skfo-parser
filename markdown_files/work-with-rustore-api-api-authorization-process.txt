# Авторизация и принципы работы в RuStore API

Для получения доступа к использованию возможностей API RuStore пройдите
авторизацию. Для этого сгенерируйте ключевую пару (открытый и закрытый ключи)
в [RuStore Консоли](https://console.rustore.ru/sign-in) и получите JWE-токен.

## Генерация ключевой пары​

примечание

Для генерации ключей используется алгоритм шифрования RSA.

  1. Откройте [RuStore Консоль](https://console.rustore.ru/sign-in).
  2. Перейдите на вкладку «Компания» в верхней части экрана в аккаунте для юридических лиц. Для физических лиц эта же вкладка называется «Разработчик».
  3. В левом боковом меню выберите «API RuStore».
  4. Нажмите «Создать ключ».

![](https://www.rustore.ru//help/assets/images/create-a-
key-6de62dfb2176bee352c1326b92b6018f.png)

  5. Введите название ключа — не более 255 символов.

![](https://www.rustore.ru//help/assets/images/parameters-
key-80edf93ec2c46426abd150d9660126b4.png)

  6. Выберите приложения, на которые будет распространяться приватный ключ.

примечание

Вы можете выбрать одно или несколько приложений, а также все сразу. Если при
генерации ключа вы поставите галочку напротив пункта «Все приложения», а затем
создадите новое приложение, ключ не будет распространяться на него.

![](https://www.rustore.ru//help/assets/images/select-
app-b3c286006f2131f99251a7f606132800.png)

  7. Выберите один или несколько методов RuStore API. Вы можете выбрать как методы приложений, к которым будет иметь доступ приватный ключ, так и общие. Общие методы будут передавать данные всех приложений вне зависимости от того, для каких приложений настроен ключ.

![](https://www.rustore.ru//help/assets/images/methods-
afb690a03b210d94adc97cb43dee0cbe.png)

![](https://www.rustore.ru//help/assets/images/general-
methods-2a573da297aa96f27a676e2f6928d7f7.png)

  8. Нажмите «Сгенерировать ключ».

![](https://www.rustore.ru//help/assets/images/generate-
key-d9a78adffa998c770b73a5f2565cb451.png)

  9. Приватный ключ покажется во всплывающем окне «Приватный ключ», скопируйте и сохраните его. Публичный ключ формируется автоматически при генерации приватного ключа и независимо от пользователя записывается в базу данных. Для авторизации понадобится только значение сгенерированного приватного ключа.

к сведению

Генерация ключевой пары доступна только владельцам компании — см. [Роли
пользователей](/help/developers/developer-account/user-roles).

При наличии пары ключей созданный приватный ключ появится в таблице в [RuStore
Консоли](https://console.rustore.ru/sign-in). В ней указаны название и ID
ключа, выбранные методы и приложения, а также когда ключ был изменён (дата
создания или обновления).

  1. Откройте [RuStore Консоль](https://console.rustore.ru/sign-in).
  2. Перейдите на вкладку «Компания» или «Разработчик» в верхней части экрана.
  3. В блоке с названиями ключей найдите необходимый ключ.
  4. Нажмите на три точки.
  5. Нажмите «Обновить».
  6. Публичный ключ обновится автоматически.

Вы также можете удалить пару ключей в [RuStore
Консоли](https://console.rustore.ru/sign-in).

  1. Откройте [RuStore Консоль](https://console.rustore.ru/sign-in).
  2. Перейдите на вкладку «Компания» или «Разработчик» в верхней части экрана.
  3. В блоке с названиями ключей найдите необходимый ключ.
  4. Нажмите на три точки.
  5. Нажмите «Удалить».

## Получение токена​

Используйте сгенерированный закрытый ключ для получения JWE-токена с помощью
метода `POST` [`/public/auth/`](/help/work-with-rustore-api/api-authorization-
token).

примечание

Срок действия JWE-токена — 900 секунд, затем его необходимо получить заново,
повторив метод `POST` [`/public/auth/`](/help/work-with-rustore-api/api-
authorization-token).

## Авторизация пройдена​

Полученный JWE-токен позволяет использовать возможности API RuStore. Для этого
его значение нужно передавать в раздел «Авторизация» с помощью `API-key`.
